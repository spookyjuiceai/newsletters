This Week's Deep Dive: OpenClaw Memory

Research

The OpenClaw memory ecosystem is more complex — and more capable — than most people realize. Most documentation stops at four basic methods. But production-grade persistent agents layer five or more techniques simultaneously. Here's the architecture that actually matters:

░░ OPENCLAW MEMORY STACK ░░ (deepest → most ephemeral)
7
Episodic/Semantic Knowledge Graph (SQLite + FTS5) Persistent
6
Curated Long-Term Memory (MEMORY.md) Durable
5
Daily Append-Only Log (memory/YYYY-MM-DD.md) 7-day TTL
4
Hybrid Vector Search (BM25 70% + Semantic 30%) Indexed
3
External Tool Memory (Mem0, Cognee, Graphiti) Plugin
2
Conversation Context Window Session
1
Ephemeral Working Memory Volatile

The feature most people miss: Pre-Compaction Memory Flush — when a session nears auto-compaction, OpenClaw triggers a silent agentic turn that writes durable memory before context is compacted. Controlled via agents.defaults.compaction.memoryFlush. It prevents the "woke up with amnesia" problem. Most users never discover it.

The community has gone far beyond the basics. Production setups run 3–5 techniques simultaneously: file-based persistence, hybrid BM25+vector search, pre-compaction flush hooks, knowledge graph engines (Cognee, Graphiti), and cognitive plugin suites with names like Nightshift, Crystallization, and Contemplation.

We're building the SpookyJuiceMemoryEngine™ — a modular, plug-in memory layer designed to solve the memory problem across agent runtimes, not just OpenClaw. Full architecture drop coming in Issue #2.
