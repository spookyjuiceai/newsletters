Deep Dive: The Install Path

Research

The goal was simple: get OpenClaw running as a persistent AI gateway on the phone. The path was not. Here's the architecture we landed on:

░░ WORKING INSTALL STACK ░░ (bottom → top)
# Layer 1: Android 16 (Pixel 10a hardware)
# Layer 2: Termux v0.118.3 (terminal emulator, no root)
# Layer 3: proot-distro Ubuntu (userspace syscall interception)
# Layer 4: NodeSource Node.js 22+ (glibc-compatible)
# Layer 5: OpenClaw 2026.3.2

pkg install -y proot-distro
proot-distro install ubuntu
proot-distro login ubuntu

# Inside Ubuntu:
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt install -y nodejs
npm config set cache /tmp/npm-cache
npm install -g openclaw

The key insight: you can't use Termux's native environment for anything with native C modules. Termux uses Android's Bionic libc instead of GNU glibc, and build tools like make break in ways that are genuinely surreal — running make dumps its help text instead of building because the argument parser is fundamentally different.

The fix is proot-distro — a userspace chroot that runs full Ubuntu inside Termux via ptrace syscall interception. No root. ~500MB disk. Full glibc. Everything just works.

Defeating Android's 5 Kill Layers

Infrastructure

Android does not want you running background processes. It has five separate layers of power management, each independently capable of killing your app. For 24/7 operation, you have to defeat all of them:

░░ ANDROID POWER MANAGEMENT LAYERS ░░
5
Doze — defers network, jobs, alarms deviceidle whitelist
4
App Standby — throttles "unused" apps RUN_IN_BACKGROUND
3
Adaptive Battery — ML-based kill prediction Unrestricted setting
2
CPU Freeze — kernel suspends processor Termux Wake Lock
1
WiFi Sleep — disconnects when screen off wifi_sleep_policy 2

The ADB commands are copy-paste ready in the full guide on GitHub. Five commands from your Mac, two manual toggles on the phone, and Android stops trying to kill your process. Every command has a verification step so you can confirm each layer is actually disabled.

The Error Log: 8 Failures, 8 Fixes

Debug

We hit every wall so you don't have to. Here's the full error log from the install session, with root causes and fixes:

ERROR 01

npm error syscall spawn git

npm needs git to resolve dependencies. Termux ships without it. Fix: pkg install -y git

ERROR 02

CMake does not seem to be available

koffi (FFI library) needs to compile from source — no prebuilt binary for Android. Fix: pkg install -y cmake make clang

ERROR 03 — THE WALL

Failed to run build step — make dumps help text

Termux's make binary parses arguments differently than GNU make. Bionic libc ≠ glibc. Hardcoded paths mismatch. Fix: Abandon native Termux. Use proot-distro Ubuntu.

ERROR 04

ENOENT: rename in npm cache

proot can't emulate cross-filesystem rename() syscall. npm atomically renames from temp to cache across FS boundaries. Fix: npm config set cache /tmp/npm-cache — same FS, no cross-FS rename.

ERROR 05

Node.js version too old

Ubuntu default repos ship ancient Node.js. OpenClaw requires ≥ 22.12. Fix: Install via NodeSource setup_22.x.

ERRORS 06–08

Screen timeout, process killing, SecurityException

Screen goes black during install → svc power stayon usb. Background processes killed → whitelist + wakelock. device_config put needs root → manual Settings toggle instead.

Total time: ~2 hours to find the working path. ~15 minutes to execute it once you know what to do. The guide has a copy-paste "happy path" at the bottom for the 15-minute version.
